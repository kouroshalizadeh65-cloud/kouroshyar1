# کوروش‌یار v3.6.35+107

نسخه اصلاحی عمیق پس از v3.6.34 با تمرکز روی هایلایت واقعی PDF، شرح پرونده کیفری، امنیت و سرعت.

## تغییرات اصلی

- جایگزینی نمایش PDF مبتنی بر `pdfx` با `syncfusion_flutter_pdfviewer` برای جستجو و هایلایت واقعی روی خود صفحه PDF.
- حذف کادر پیش‌نمایش متن جستجوشده از پایین نوار جستجو؛ نتیجه جستجو اکنون داخل خود فایل PDF هایلایت می‌شود.
- افزودن دکمه پاک کردن هایلایت PDF و حفظ نتیجه قبلی/بعدی با حرکت روی نمونه‌های هایلایت‌شده داخل PDF.
- حفظ امکان استخراج متن PDF متنی برای کپی، با پیام واضح برای PDF اسکن‌شده بدون OCR.
- اصلاح منطق شرح پرونده کیفری: جمله وضعیت پرونده بر اساس آخرین اقدام واقعی ساخته می‌شود، نه صرفاً وجود کیفرخواست در تاریخچه.
- اگر بعد از صدور/ارسال کیفرخواست، رای، ابلاغ رای، قطعیت رای، تجدیدنظر، فرجام، اعاده دادرسی یا اجرای احکام ثبت شده باشد، شرح کیفری دیگر به اشتباه نمی‌نویسد پرونده در مرحله رسیدگی دادگاه کیفری است.
- تقویت پنل مشخصات پرونده: باز بودن پیش‌فرض، حذف اتکای قبلی به CrossFade و استفاده از AnimatedSize برای باز/بسته شدن پایدارتر.
- حذف dependency `pdfx` و کاهش وابستگی اضافه برای سبک‌تر شدن build و کاهش ریسک conflict.
- بهینه‌سازی ذخیره دیتابیس محلی با JSON فشرده و نوشتن روی فایل موقت قبل از جایگزینی فایل اصلی.
- حذف keystore از ZIP سورس و حذف رمزهای hardcoded از GitHub Actions.
- اصلاح workflow برای استفاده از GitHub Secrets جهت امضای ثابت APK؛ در صورت نبود secrets، فقط برای تست build یک keystore موقت CI ساخته می‌شود.
- به‌روزرسانی صفحه درباره برنامه، تنظیمات و نام APK به نسخه 3.6.35.

## GitHub Secrets پیشنهادی برای امضای ثابت APK

برای اینکه APKهای بعدی روی APK قبلی نصب/آپدیت شوند، این secrets را در GitHub تنظیم کنید:

- `KOUROSHYAR_KEYSTORE_BASE64`
- `KOUROSHYAR_KEYSTORE_PASSWORD`
- `KOUROSHYAR_KEY_PASSWORD`
- `KOUROSHYAR_KEY_ALIAS`

اگر این موارد تنظیم نشوند، GitHub Actions برای تست build، keystore موقت می‌سازد؛ اما امضای APK پایدار نیست و ممکن است برای نصب نسخه بعدی لازم باشد نسخه قبلی حذف شود.

## نکات تست ضروری

- جستجوی عبارت در PDF متنی و مشاهده هایلایت داخل خود صفحه PDF.
- حرکت نتیجه قبلی/بعدی داخل PDF.
- پاک کردن هایلایت PDF.
- تست PDF اسکن‌شده و نمایش پیام نیاز به OCR.
- ثبت تاریخ صدور کیفرخواست، سپس ثبت تاریخ صدور رای، و بررسی اینکه شرح کیفری دیگر مرحله پرونده را کیفرخواست/دادگاه کیفری اعلام نکند.
- باز و بسته کردن چندباره مشخصات پرونده.
- اجرای GitHub Actions و بررسی `flutter analyze` و build.

## محدودیت تست در این محیط

در محیط ChatGPT، Flutter/Dart نصب نبود؛ بنابراین `flutter analyze` و `flutter build` محلی اجرا نشد. بررسی‌ها به‌صورت کدخوانی، اصلاح سورس و کنترل ساختار ZIP انجام شد.
