# کوروش‌یار v3.6.10

## اصلاح امنیتی ریشه‌ای

- اضافه شدن install guard برای تشخیص دیتابیس restoreشده از نصب قبلی.
- استفاده از مسیر Android no-backup برای نگهداری شناسه نصب فعلی.
- اگر دیتابیس restoreشده با شناسه نصب فعلی همخوان نباشد، دیتابیس و فایل‌های حساس پاک می‌شوند.
- اگر دیتابیس legacy بدون install guard ولی همراه با قفل/رمز/بیومتریک برگردد، به عنوان restore ناامن پاک می‌شود.
- رمز و اثر انگشت ذخیره‌شده نسخه‌های قبل نامعتبر شدند و باید در نصب فعلی دوباره فعال شوند.
- بکاپ خودکار داخلی خاموش شد؛ پشتیبان فقط با انتخاب صریح کاربر ساخته می‌شود.
- متن‌های بخش حریم خصوصی و پشتیبان‌گیری اصلاح شدند.
